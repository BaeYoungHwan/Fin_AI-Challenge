## 1. 개요

보이스피싱·이상금융거래 의심 상황을 입력하면 위험 신호를 분석하고, 법령 근거와 실전 대응절차를
안내하는 AI 금융보안비서. 2026 금융 AI Challenge(금융보안원 주최) 제출용 MVP.

## 2. 문제 정의

보이스피싱·메신저피싱 등 이상금융거래 피해는 대부분 "판단이 필요한 순간"에 발생한다. 피해자는
의심스러운 상황(문자, 전화, 메신저 요청 등)을 접했을 때 이것이 실제 위험인지, 어떤 법적 보호장치가
있는지, 지금 무엇을 해야 하는지를 즉시 알기 어렵다. 기존 금융감독원·경찰청 자료는 정적이고 분산돼
있어 상황에 맞는 즉각적인 안내를 받기 어렵다. 특히 디지털 취약계층(고령층 등)은 이 판단 과정에서
더 취약하다.

## 3. 타겟 유저

| 페르소나 | 설명 | 주요 시나리오 |
|----------|------|---------------|
| 주 사용자 | 보이스피싱·이상금융거래 의심 상황에 처한 일반 금융소비자 | 가족·지인 사칭 문자로 급전 요청을 받고 진위를 판단하고 싶음 |
| 부 사용자 | 디지털 취약계층(고령층, 정보 접근성이 낮은 사용자) | 기관 사칭 전화를 받고 불안한 상태에서 즉시 행동요령이 필요함 |

## 4. 성공 지표 (KPI)

- 법령 인용 환각(근거 없는 조문 인용) 발생률 최소화 (환각 검증 루프로 재분석 처리)
- 큐레이션된 사기 유형 커버리지 — 주요 유형(가족·지인 사칭형, 기관 사칭형, 대출빙자형 등) 5종 이상
- 사용자 피드백 기준 "행동요령이 실제 도움됨" 비율

## 5. 기술 스택

Python 3.12 + LangChain + 로컬 벡터 저장소(numpy+JSON, 사기유형·대응절차 큐레이션 DB) + 법제처 Open API(법적 근거
실시간 검색) + **Groq API(무료 티어, Llama 3.3 70B)** + Streamlit UI.

## 6. 배포 환경

**Streamlit Community Cloud (무료 배포)**. 제약조건: 웹서비스 URL이 2026-09-07 11:00 ~ 09-11 23:59
상시 접근 가능해야 함 — GitHub 레포 연결만으로 배포되고 서버 관리가 불필요해 이 제약을 가장 낮은
공수로 충족.

## 7. MVP 핵심 기능

- [ ] 의심 상황(문구/스크립트/상황 설명) 텍스트 입력
- [ ] 위험 신호(red flag) 탐지 — 매체, 요청 유형, 본인확인 절차 언급 여부 등 패턴 기반
- [ ] 사기 유형 분류 — 큐레이션된 유형 DB(가족·지인 사칭형, 기관 사칭형, 대출빙자형 등)와 대조
- [ ] 법제처 API 기반 관련 법령 실시간 검색 + 인용 (환각 검증 포함 — 미제공 조문 인용 시 재분석)
- [ ] 위험도 안내 + 상황과 무관하게 항상 유효한 안전 행동요령 제공 (단정적 판정 금지)
- [ ] 사용자 피드백 수집 → 유형 DB 축적

## 8. MVP 제외 사항 ⚠️

> Claude는 아래 항목을 임의로 구현하지 않습니다.

- 실시간 은행/통신사 시스템 연동을 통한 실제 이상거래 탐지
- 신고 자동 접수(경찰청/금융감독원 시스템 직접 연동)
- 다국어(i18n) 지원
- 사기 여부에 대한 단정적(예/아니오) 판정 제공

## 9. 제약사항

  않는다 — 재사용은 범용 아키텍처 패턴(RAG + 환각 검증 + 피드백 루프) 수준으로 제한.

- 개인 프로젝트 — 운영 복잡도 최소화, 개발 공수 최소화 우선.
