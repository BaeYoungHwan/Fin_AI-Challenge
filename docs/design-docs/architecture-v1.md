## 1. 시스템 개요

사용자가 보이스피싱·이상금융거래가 의심되는 상황(문자, 전화 스크립트, 메신저 요청 등)을 입력하면,
위험 신호를 탐지하고 법제처 API로 검색한 법령 조문 + 큐레이션된 사기 유형 사례를 근거로 위험도를
안내하며, 상황과 무관하게 항상 유효한 안전 행동요령을 제공하는 3-Agent 파이프라인 웹서비스.

## 2. 컴포넌트 다이어그램

```
상황 입력 (Streamlit UI)
    │
    ▼
[Retrieval Agent]  ← 법제처 Open API (실시간 법령 검색)
  위험 신호(red flag) 패턴 탐지 → 적용 법령 키워드 결정
  ← 로컬 벡터 저장소 (큐레이션 사기 유형·대응절차 Top-3)
    │
    ▼
[Analysis Agent]   ← Groq API (무료 티어, Llama 3.3 70B)
  위험도(주의 수준) 판단 + 법령 조항 인용 + 안전 행동요령 생성
  [검증 단계] 인용 조문 환각 탐지 → 미제공 법령 인용 시 자동 재분석
  ※ 사기 여부 단정 금지 — 항상 "위험도 + 행동요령" 형태로 출력
    │
    ▼
[Feedback Agent]   ← 사용자 피드백(도움됐는지) 입력
  품질 필터링 → 유효 피드백만 로컬 벡터 저장소 저장 → 사기 유형 DB 축적
```

## 3. 레이어 구조

```
src/
├── agents/
│   ├── retrieval_agent.py    # 법령 검색 + 위험 신호 탐지 + 유사 유형 조회
│   ├── analysis_agent.py     # 위험도 판단 + 행동요령 생성 (LLM) + 환각 검증
│   └── feedback_agent.py     # 피드백 품질 필터링 + 유형 DB 저장
├── tools/
│   ├── law_search.py         # 법제처 API 래퍼
│   └── scam_classifier.py    # 위험 신호 패턴 + 사기 유형 매핑
├── db/
│   └── vector_store.py       # 로컬 벡터 저장소 관리 (numpy+JSON, 사기 유형·대응절차)
└── ui/
    └── app.py                # Streamlit UI
scripts/
└── seed_chroma.py            # 사기 유형·대응절차 시드 데이터 적재
tests/
├── test_smoke.py             # P0 스모크 테스트
└── test_scenarios.py         # 사기 유형별 E2E 테스트
```

## 4. 데이터 흐름

**골든 패스 예시**: "가족이 메신저로 100만원을 급하게 빌려달라고 함"

1. Retrieval Agent: "메신저", "급전", "이체" 키워드 탐지 → 통신사기피해환급법·전자금융거래법 검색,
   로컬 벡터 저장소에서 "가족·지인 사칭형" 유사 유형 조회
2. Analysis Agent: 위험 신호(본인확인 절차 언급 없음 등) 종합 → "주의 필요" 수준 안내, 법령 조문
   인용(환각 검증 통과), 안전 행동요령("직접 전화로 재확인", "의심 시 112/1332 신고") 생성
3. Feedback Agent: 사용자가 "실제 도움됨" 피드백 입력 → 품질 필터 통과 시 로컬 벡터 저장소에 저장

## 5. 주요 설계 결정

| 결정 | 선택 | 이유 | ADR |
|------|------|------|-----|
| 판정 방식 | 위험도 + 안전행동 병행, 단정 금지 | 오판 리스크 최소화 | - |
| 법적 근거 | 법제처 API 실시간 검색 | 공식·최신 데이터, 환각 방지 검증 가능 | - |
| 실무 절차 | 수작업 큐레이션 + 로컬 벡터 저장소 | 법령에 없는 신고 절차·전화번호 등 보완 | - |

## 6. 비기능 요건 달성 전략

- **환각 방지**: Analysis Agent의 인용 검증 단계에서 법제처 API가 반환하지 않은 법령명이 등장하면
  자동 재분석 프롬프트로 재시도
- **가용성**: Phase 0에서 로컬 Ollama 대신 클라우드 배포/호스팅 API로 전환하여 제출 기간 상시 접근
  보장
- **성능**: LLM 스트리밍 응답으로 체감 지연 최소화

## 7. 보안 고려사항

- 사용자가 입력하는 상황 설명에는 개인정보(계좌번호, 연락처 등)가 포함될 수 있음 — 저장 시 최소
  수집 원칙, 불필요한 민감정보 로깅 금지
- `LAW_API_KEY` 등 API 키는 `.env`로 관리, 커밋 금지
